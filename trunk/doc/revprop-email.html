Using Subversion properties to email revision property change details

Using properties in partnership with the post-revprop-change hook and svnlook, we can provide a flexible, granular, visible and version controlled mechanism for emailing revision property change details.

property: hook:revprop-email

This property contains a line delimited list of email addresses. It is used by the post-revprop-change hook to determine to whom email should be sent.

hook:revprop-email can exist on the directory or parent directories of files or directories you wish to monitor.

e.g. To send an email displaying the author, revision, revision property and new property value to watchers@example.com, the hook:revprop-email property is set to the following value on the repository root:

watchers@example.com

hook: post-revprop-change

This hook script should call a program which searches the HEAD revision for hook:revprop-email properties on or above the directories changed. The svnlook dirs-changed command can be used in conjunction with the revision number passed to post-commit to retrieve list of modified directories.

If matching properties are found, the hook script will:

  1. Get property values, parsing each email address
  2. Collate email addresses, removing duplicates
  3. Send email(s) containing author, revision, revision property modified, and the new property value
